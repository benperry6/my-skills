# Integration Boundaries

This skill is intentionally narrow.

## my-personal-harness-engineering

Use that skill to decide whether an evaluator is needed at all.

Use this skill after that choice has already been made.

## my-personal-second-opinion

Use `my-personal-second-opinion` for:

- cross-engine technical review
- plan challenge
- independent code-level audit

Use this skill for:

- fulfillment-focused QA
- behavior-first acceptance
- strict acceptance verdicts

The two can be combined, but they are not the same role.

## challenge

Use `challenge` for adversarial code review and devil's-advocate critique.

This skill is not trying to break the change intellectually.
It is trying to decide whether the requested job was actually completed and proven.

## my-personal-subagent-orchestration

That skill handles worker mechanics.

This skill handles evaluator judgment.

## my-personal-verified-learning-loop

Use it when the evaluator itself needs calibration updates after real usage.
