# Vendor Priority

This file exists to stop premature vendor sprawl.

## Canonical priority

Unless the context clearly proves otherwise, open vendors in this order:

1. Google
2. Meta
3. Any additional vendor only if justified by the real channel plan (TikTok, LinkedIn, Pinterest, Reddit, Snap, Microsoft, etc.)
4. Everything else later, only if justified

## Why this order

- Google is foundational for analytics, intent capture, and future ads integration
- Meta is usually the highest-value cross-project paid social foundation
- TikTok is highly useful in discovery-led contexts but should not be privileged over every other channel-fit vendor
- The rest can be opened early when the business model and actual channel plan clearly require them
- The real constraint is relevance and QA capacity, not an artificial "phase 2 only" rule

## By archetype

## SaaS

- Default: Google, then Meta
- Add TikTok from day one if the product has clear broad-consumer or creator/discovery fit
- Add LinkedIn from day one if the business is clearly B2B and paid LinkedIn is actually on the roadmap
- Add Microsoft from day one if paid search beyond Google is already part of the near-term plan
- Add Reddit from day one if the acquisition motion is community-led and the product fits those audiences

## E-commerce

- Default: Google, then Meta
- Add TikTok from day one if the category is discovery-led
- Add Pinterest from day one if the catalog and creative format support it
- Add Snap from day one if the audience and creative format clearly fit
- Add Microsoft from day one if search demand and shopping/search expansion beyond Google are already planned

## Affiliate / Content

- Default: Google first
- Add Meta if remarketing or audience building is realistic
- Add TikTok from day one if the content model genuinely fits short-form discovery
- Add Pinterest from day one if the distribution model is visual search or save-driven
- Add Reddit from day one if the traffic plan depends on community-native distribution
- Add Microsoft from day one if search monetization beyond Google is already part of the plan

## Anti-patterns

- Opening eight vendors before having a stable event model
- Installing a platform just because it exists, with no channel intent behind it
- Treating every optional vendor as mandatory just because it is allowed from day one
- Letting vendor count outrun QA capacity

The bias of this skill is disciplined rollout, not maximal platform count from day one.
