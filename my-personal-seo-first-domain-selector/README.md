# my-personal-seo-first-domain-selector

Choose the best **SEO-first domain** from real search demand, not from branding taste.

This skill exists for a specific idea:

Instead of starting from an unknown brand and hoping Google learns it later, start from a keyword people already search for, rank on it, satisfy intent strongly, and let Google progressively associate that keyword with the site in a more brand-like way.

## The core theory

The theory behind this skill is:

1. choose a domain name that is also a searched keyword, or a very close quasi-EMD
2. capture traffic faster because the domain matches a real query
3. if the page ranks and satisfies intent, usage signals improve
4. Google can start associating that keyword with the site more strongly over time
5. a generic search term can gradually become an authority-building asset instead of just raw traffic

In one sentence:

> Instead of starting from an unknown brand, take an EMD on a searched keyword, rank on it, satisfy users, and let Google increasingly treat that keyword as if it points to your site.

## What this skill is for

Use this skill when the real question is:

- which keyword should become the domain
- which SEO-first domain best fits the business model
- whether the highest-volume keyword is actually relevant enough
- whether Google’s SERP intent matches the offer closely enough
- whether the final domain candidates are really available

This is **not** a branding skill.

It does not try to invent a catchy name first. It tries to find the best **keyword-domain vehicle** for SEO.

## The important nuance

The theory is not “pick the biggest keyword and win.”

The real target is:

- the highest **relevant** search volume
- for the right offer
- with the right intent
- on a SERP that already accepts the type of page or operator you want to be

If the keyword is too broad, too informational, or attracts the wrong audience, the theory breaks:

- users bounce or disengage
- signals weaken
- Google does not build the right association between query and site

So this skill optimizes for **relevant volume**, not raw volume.

## Why SERP validation is mandatory

This is the most important lesson behind the skill.

Keyword tools can generate strong candidates, but they cannot tell you with enough certainty whether Google sees the query the same way your business does.

That is why the skill performs a real SERP review before making a final recommendation:

- inspect the top results
- inspect the key pages behind them
- reverse-engineer the intent Google is rewarding
- reject candidates whose SERP is materially misaligned with the offer

The MOVA case is the clearest example:

- `vendre sa maison` had more volume
- but its SERP was dominated by informational guides and large portals
- `vendre sa maison rapidement` had less volume
- but its SERP already included actors much closer to the real business model

The winner was not the keyword with the biggest volume.
The winner was the keyword whose **volume, business fit, and SERP fit** worked together.

## What the skill actually does

The skill follows this sequence:

1. read the business and project context first
2. define the real SEO job to be done
3. use the right keyword data source for the market
4. build and rank keyword-domain candidates
5. make the trade-off between volume and precision explicit
6. perform strict SERP validation on the best candidates
7. verify real domain availability with the dedicated registrar-backed checker
8. output a ranked shortlist and a clear winner

## When to use it

Use it when you want:

- a best-domain recommendation for SEO
- an EMD or quasi-EMD decision
- a shortlist of keyword-based domains
- a data-backed choice between multiple domain candidates
- a final decision that includes real availability checks

## When not to use it

Do not use it when you want:

- pure branding or naming exploration
- creative or memorable brand ideas
- trademark clearance
- a simple fixed-list availability check

For the broader upstream work, this skill is meant to sit **after** market and keyword mapping.
Its natural companion is:

- `my-personal-seo-market-mapper`

That upstream skill owns:

- business context distillation
- market language discovery
- keyword research
- clustering
- strategic SEO synthesis

Then this skill uses that material to make the domain decision.

## Practical output

A good run of this skill should produce:

- one clear winner
- a ranked top 10
- explicit trade-offs
- SERP-backed reasoning
- availability-backed validation

The final result should answer:

**“What is the best SEO-first domain for this business, according to the EMD theory, real search demand, real SERP intent, and real domain availability?”**
