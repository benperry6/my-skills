# Integration Boundaries

This skill is intentionally not a god-skill.

Use it to choose and shape the harness.
Use the adjacent skills for their narrower jobs.

## my-personal-second-opinion

Use for:

- external code-level evaluation
- plan challenge
- post-implementation independent audit
- cross-engine truth seeking

This harness skill decides whether such an evaluator is needed.
`my-personal-second-opinion` performs that evaluator job.

## my-personal-subagent-orchestration

Use for:

- durable supervision of delegated workers
- run registries
- inactivity thresholds
- recovery and respawn rules
- detached worker recovery across sessions

This harness skill decides whether the task has grown large enough to need durable worker orchestration.
`my-personal-subagent-orchestration` handles those mechanics.

## my-personal-verified-learning-loop

Use for:

- turning real harness outcomes into runtime or verified learnings
- promoting or refusing to promote new doctrine
- updating the smallest correct surface after real evidence

This harness skill defines the operating doctrine.
`my-personal-verified-learning-loop` governs how that doctrine changes after real usage.

## Boundary summary

This skill owns:

- harness choice
- role boundaries
- artifact doctrine
- context reset doctrine
- simplification doctrine

It does not own:

- provider-specific session recovery
- cross-engine execution mechanics
- generic self-modification policy
